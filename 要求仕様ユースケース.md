```mermaid   
sequenceDiagram
    autonumber

    actor Admin as 管理者
    participant UI as Admin UI（Next.js）
    participant Hooks as Hooks層
    participant API as FastAPI

    participant DB as SQLite DB

    %% --- レポート生成開始 ---
    Admin ->> UI: 「レポート生成」ボタン
    UI ->> Hooks: useGenerateReport()
    Hooks ->> API: POST /reports/{id}/generate
    API ->> DB: タスク作成（status=pending）
    API -->> Hooks: task_id

    %% --- タスク進捗監視 ---
    loop 進捗確認
        UI ->> Hooks: useTaskStatus(task_id)
        Hooks ->> API: GET /tasks/{task_id}
        API ->> DB: 進捗取得
        DB -->> API: status, progress, message
        API -->> Hooks: 進捗データ
        Hooks -->> UI: 進捗バー更新
    end

    %% --- 非同期タスク処理 ---
    API ->> Task: タスク開始

    Task ->> DB: コメント取得
    Task ->> LLM: クラスタリング
    LLM -->> Task: クラスタ結果
    Task ->> DB: 保存（clusters）

    Task ->> LLM: クラスタ要約
    LLM -->> Task: 要約
    Task ->> DB: 保存（summaries）

    Task ->> LLM: 全体要約
    LLM -->> Task: 全体要約
    Task ->> DB: 保存（global_summary）

    Task ->> LLM: 章構造生成
    LLM -->> Task: 章構造
    Task ->> DB: 保存（sections）

    Task ->> DB: タスク完了（status=done）
``` 

市民向けユースケース図（Mermaid）

```mermaid
%% 市民向けユースケース図（広聴AI public-viewer）

flowchart LR

    %% アクター
    Citizen([市民])

    %% システム境界
    subgraph PublicViewer[広聴AI 公開レポート（public-viewer）]
        UC1[レポートのトップページを見る]
        UC2[全体要約を読む]
        UC3[テーマ（クラスタ）を選ぶ]
        UC4[テーマの要約を読む]
        UC5[市民のコメントを読む]
        UC6[グラフで全体像を把握する]
        UC7[章（セクション）を読む]
        UC8[レポート全体の総括を読む]
    end

    %% 関係
    Citizen --> UC1
    Citizen --> UC2
    Citizen --> UC3
    Citizen --> UC4
    Citizen --> UC5
    Citizen --> UC6
    Citizen --> UC7
    Citizen --> UC8
```


---

図の読み方（市民向けの行動フロー）

• UC1：公開されたレポートページにアクセス
• UC2：AI がまとめた全体要約を読む
• UC3：興味のあるテーマ（クラスタ）を選ぶ
• UC4：テーマの要約を読む
• UC5：実際の市民コメントを読む
• UC6：散布図や棒グラフで全体像を理解
• UC7：章（セクション）ごとに深掘り
• UC8：レポート全体の総括を読む


市民はこの流れで、
「全体 → テーマ → コメント → グラフ → 章 → 総括」
という自然な理解プロセスを辿ります。
